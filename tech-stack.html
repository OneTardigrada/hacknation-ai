Architecture & Tech Stack

From the three user-facing surfaces down to the LLM and on-device privacy layer.

1 · Surfaces

Three screens, one brain

What the user, the merchant and stakeholders actually see.
Consumer Phone
Single offer card, swipe-to-redeem, QR + cashback, FOMO countdown.
Merchant Tablet
Rule configurator, pre-order queue, quiet-hour chart, share-count.
City Boardgame Map
MapLibre view of geofences, merchants, live offer pulses.
Wallet & Friends
Wallet ID, transaction history, friends layer, share codes.

3 · API · Route Handlers

Next.js app/api/*

Streaming endpoints between phone, tablet and the AI core.
/api/generate-offer
Streamed JSON LLM offer generation.
/api/merchant-stats
Quiet/busy hours, redemptions, share-count.
/api/preorder
Pre-order intent → merchant queue.
/api/redeem
QR redemption + cashback finalisation.
/api/wallet-id
Pseudonymous wallet token issuance.
/api/weather
OpenWeatherMap proxy + mock fallback.
4 · Domain Logic · lib/

The context pipeline

Pure TS modules — typed, testable, deterministic.
context-engine.ts
Merges weather + hour + distance + goal → ContextSnapshot.
geofence.ts
Haversine distance, radius rings, enter/exit triggers.
offer-prompt.ts
Renders system+user prompt with strict JSON schema.
friends.ts
Opt-in social graph, share-code propagation.
wallet-id.ts
Pseudonymous token — never PII to merchants.
merchant-sim.ts
Live foot-traffic simulation for the boardgame.
scenarios.ts · personas.ts
Demo presets, time-machine, persona switching.
preorder.ts
Pre-order state machine, queue + ETA.
5 · AI Core

Streaming generation

Cloud LLM today, on-device SLM path tomorrow.
OpenAI · GPT
Streaming JSON tokens → progressive offer card.
useStreamingOffer hook
Parses partial JSON, renders headline → discount → timer.
slm-layer.ts
On-device small-language-model path for full privacy.
Deterministic fallback
No API key? Demo still runs — rule-based offer pipeline.
6 · Privacy & Trust

Bank-grade by design

Consent-first, GDPR-konform, pseudonymous, on-device-ready.
Consent toggles
Location, weather, transactions, friends — each opt-in.
Pseudonymous Wallet ID
Merchants see a token, never a name.
Data minimisation
Only 4 signals reach the LLM — no PII, no history.
Right to erasure
One tap clears history + share-graph.
